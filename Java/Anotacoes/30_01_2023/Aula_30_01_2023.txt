Pode configurar global, qualquer pasta que abrir está configurado a sua conta do git, mas por ser um pc da escola, configura LOCALMENTE


-> Repositório local
	Pasta (ou diretório) na máquina do programador, onde são gravadas as versões

-> Repositório remoto
	Pasta ou diretório na nuvem, na qual é gravado o projeto completo, com todas suas versões

-> Staging <-
	Local intermediário entre a máquina do desenvolvedor e o repositório remoto, para revisão de alterações

-> Commit
	Ato de registrar ou salvar as alterações

-> Ramo Principal <-
	Sequência principal do código-fonte

-> Branch <-
	Uma ramificação do ramo principal, para testes em geral
	Tipo um teste paralelo, se funcionar, joga no ramo principal, se não, descarta

-> README
	Arquivo que é uma mescla de cartão de visitas e documentação do projeto

-> .gitignore <-
	arquivo de texto que guarda os arquivos ou pastas que devem ser ignorados, como o básico pro projeto rodar por exemplo, como não é algo que muitas vezes não será alterado, não precisa "perder tempo" upando esses arquivos

-> 'git config --local user.name "LoriaLawrenceZ" '
	Configurando que o usuário LOCAL terá esse nome

-> 'git config --local user.email "lorenzozimbresfilmmaker@gmail.com" '
	Configurando que o usuário local terá esse email

-> 'git branch -M main'
	Muda o ramo de master pra main

-> 'git add .'
	adiciona todos os arquivos pro staging
	'.' - seleciona todos os arquivos do repositório local

-> 'git commit -m "30/01/2023" '
	Faz o commit
	'-m' é a mensagem do commit

------------------

-> Cria um repositório no Github

-> 'git remote add origin https://github....git'
	'origin' serve como pseudônimo para não ter que escrever toda a URL toda hora
	linkando a pasta local com a pasta remota

-> 'git remote'
	mostra se já tem alguma pasta linkado remotamente (quase certeza)

-> 'git push -u origin main'
	Envia versões de arquivos pra sua conta no github
	No caso, enviou pra origin (remoto) a branch main
